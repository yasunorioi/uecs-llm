# Pico 2 W アクチュエータノード基板 設計ドキュメント

**バージョン**: 1.0
**作成日**: 2026-02-08
**ターゲット**: Raspberry Pi Pico 2 W
**用途**: 4チャンネルリレー駆動（電磁弁・換気扇制御）

---

## 1. 概要

本基板は、Raspberry Pi Pico 2 W を使用した4チャンネルリレー駆動基板である。温室制御盤内に設置し、WiFi経由でMQTT通信を行い、電磁弁・換気扇等のAC負荷を制御する。

### 主な特徴
- **4chリレー駆動**: 電磁弁、換気扇、灌水バルブ等を制御
- **WiFi通信**: Pico 2 W の無線LANでMQTT通信
- **USB給電**: USB 5V給電（1ポートで動作）
- **フォトカプラ絶縁**: AC負荷との電気的絶縁による安全設計
- **制御盤内設置**: 温室制御盤内の限られたスペースに対応

### 使用想定
- 温室制御システムのアクチュエータノード
- 制御盤内設置（センサーノードは圃場分散配置）
- WiFi通信範囲内（制御盤は建物近傍）
- Home Assistant / Node-RED からの制御

## 4. リレー駆動回路詳細

### 4.1 なぜMOSFET（2N7002）か？

**選定理由:**
1. **3.3Vゲート駆動可能**: Vgs(th) = 1.0V (typ), 2.5V (max)
   - 3.3V駆動で十分なドレイン電流（50mA以上）を確保
2. **ドレイン電流**: Ids = 300mA (max)
   - リレーコイル（50mA）の6倍の余裕
3. **小型パッケージ**: SOT-23
   - 基板面積の削減
4. **入手性**: JLCPCB標準部品（C8545）
   - 組立コスト削減

**代替候補:**
- BSS138 (C193658): Vgs(th)=0.8V, Ids=200mA（余裕小）
- AO3400A (C20917): Vgs(th)=0.5V, Ids=5.8A（オーバースペック）

### 4.2 なぜフォトカプラ（PC817）か？

**選定理由:**
1. **電気的絶縁**: AC100V負荷と制御回路を完全分離
   - 絶縁耐圧: 5000Vrms (IEC60747-5-5規格)
   - 感電・ノイズ混入を防止
2. **安全規格**: UL認定、VDE認定
   - 産業用途で実績あり
3. **電流伝達比**: CTR = 50-600%
   - 入力電流10mA → 出力電流 5-60mA（MOSFETゲート駆動に十分）
4. **入手性**: JLCPCB標準部品（C66723）

**代替候補:**
- TLP291 (シャープ): CTR高いが高価
- 4N35 (TI): 古い世代、入手性やや劣る

### 4.3 プルダウン抵抗（10kΩ）の役割

**必要性:**
- Pico起動時（GPIO未初期化時）にMOSFETゲートがフローティング
- フローティング状態ではリレーが不定動作（誤ONの危険）
- 10kΩプルダウンでゲートを強制的にGNDに引く → 起動時リレーOFF保証

**抵抗値計算:**
- ゲートリーク電流: < 1nA (2N7002データシート)
- プルダウン電流: 3.3V / 10kΩ = 0.33mA（無視できる）

### 4.4 フライバックダイオード（1N4148）

**必要性:**
- リレーコイルはインダクタ（L成分）
- OFF時に逆起電力（-V = L × dI/dt）が発生 → MOSFETを破壊
- フライバックダイオードで逆起電力を5V電源に還流

**選定理由:**
- 高速スイッチング: trr = 4ns
- 小型: DO-35 or SOD-323
- 入手性: JLCPCB標準部品（C81598）

## 6. 電力計算

### 6.1 リレーコイル消費電力

**1チャンネルあたり:**
- リレーコイル: 5V × 50mA = 0.25W
- インジケータLED: 3.3V × 10mA = 0.033W
- フォトカプラLED: 3.3V × 10mA = 0.033W
- 合計: **0.316W/ch**

**4チャンネル同時ON時:**
- リレー: 0.25W × 4 = 1.0W
- LED: 0.033W × 4 = 0.132W
- フォトカプラ: 0.033W × 4 = 0.132W
- Pico 2 W本体: 0.5W（WiFi通信時）
- **合計: 1.76W**

### 6.2 USB給電要件

**USB 2.0規格:**
- 標準: 500mA @ 5V = 2.5W
- USB 3.0: 900mA @ 5V = 4.5W

**余裕率:**
- 消費電力: 1.76W
- USB 2.0供給: 2.5W
- **余裕: 0.74W（29.6%）** → **OK**

**突入電流対策:**
- リレー駆動時の突入電流: < 100mA/ch（リレーコイルのL成分により緩和）
- C1（100μF）で突入電流を平滑化

## 7. 安全設計

### 7.1 電気的絶縁

**フォトカプラによる絶縁:**
- 1次側（制御側）: 3.3V（Pico GPIO）
- 2次側（負荷側）: 5V（リレー駆動）
- AC負荷: 100V（端子台経由）

**絶縁耐圧:**
- PC817: 5000Vrms（IEC60747-5-5規格）
- クリアランス: 基板設計で4mm以上確保（IEC61010-1規格）

### 7.2 起動時安全

**プルダウン抵抗による起動時OFF保証:**
- MOSFETゲート: 10kΩプルダウン
- Pico起動時（GPIO未初期化）: ゲート電圧 = 0V → MOSFET OFF → リレーOFF
- ファームウェア初期化: `machine.Pin(10, machine.Pin.OUT, value=0)` → 明示的にOFF

### 7.3 逆起電力保護

**フライバックダイオード:**
- リレーコイルOFF時の逆起電力を5V電源に還流
- MOSFETのドレイン-ソース間電圧 (Vds) を保護

### 7.4 過電流保護

**USBポリスイッチ:**
- Pico 2 W基板にはUSBポリスイッチ（復帰型ヒューズ）が実装済み
- 過電流時（> 500mA）に自動遮断

## 8. 既存設計との対応

### 8.1 config_actuator.py との整合性

| 項目 | config_actuator.py | 本基板設計 | 整合性 |
|------|-------------------|-----------|-------|
| GPIO番号 | RELAY_PINS = [10, 11, 12, 13] | GP10-GP13 | ✅ 一致 |
| 極性 | RELAY_ACTIVE_HIGH = True | HIGH = ON | ✅ 一致 |
| CCMマッピング | Irri→0, Valve→1, VenFan→2, Relay→3 | CH0-CH3 | ✅ 一致 |

### 8.2 actuator_control.py との注意点

**不整合:**
- `actuator_control.py`: デフォルト `active_low=True`
- 本基板: `RELAY_ACTIVE_HIGH = True`（HIGH = ON）

**対応方法:**
```python
from actuator_control import RelayController

# 【重要】active_low=False を明示的に指定
relays = RelayController(
    pins=[10, 11, 12, 13],
    active_low=False,  # ← この基板はactive_high設計
    names=['Irri', 'Valve', 'VenFan', 'Relay']
)
```

**推奨:**
- `config_actuator.py` と `actuator_control.py` の初期化コードを統一
- `active_low=False` をデフォルト引数として明記

## 9. 配線仕様

### 9.1 AC負荷配線

**電磁弁（AC100V, 5VA）:**
- 電流: 5VA / 100V = 50mA

- 電流: 50W / 100V = 0.5A
- 電線: VVF 1.6mm
- 最大長: 30m（電圧降下 < 2V）

**USB延長ケーブル:**

- 最大長: 3m（USB規格推奨）
- 給電: 制御盤内AC100V → USB ACアダプタ（5V/1A）
