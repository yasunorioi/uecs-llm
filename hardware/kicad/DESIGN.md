# Grove Shield for Pi Pico 代替基板 設計ドキュメント

**バージョン**: 1.0
**作成日**: 2026-02-07
**ターゲット**: W5500-EVB-Pico-PoE スタック接続

---

## 1. 概要

本基板は、M5Stack Groveエコシステムと互換性のあるI2Cセンサーを、WIZnet W5500-EVB-Pico-PoE基板に接続するためのブレークアウト基板である。

### 主な特徴
- **Grove互換**: M5Stack Grove I2Cセンサー（SHT40, SCD41, BH1750等）を直接接続可能
- **PoE給電対応**: W5500-EVB-Pico-PoEからの3.3V電源供給
- **ADC入力**: アナログセンサー（土壌水分、日射量等）対応
- **スタック接続**: 2×20ピンヘッダで上下にスタック可能

### 使用想定
- 農業IoTノード（温室環境モニタリング）
- W5500-EVB-Pico-PoE基板の上にスタック
- PoE給電でケーブル1本で動作
- MQTTでクラウドにデータ送信

## 5. I2Cプルアップ抵抗計算

### 理論計算

**最小プルアップ抵抗値（Low電圧保証）:**

```
Rp_min = (Vcc - Vol) / Iol
       = (3.3V - 0.4V) / 3mA
       = 967Ω
```

- `Vcc`: 3.3V
- `Vol`: Low出力電圧（最大0.4V）
- `Iol`: Low出力電流（3mA標準）

**最大プルアップ抵抗値（高速モード400kHz対応）:**

```
Rp_max = 1 / (0.8472 × Cb × f)
```

条件:
- `Cb`: バス容量（50pF、短配線想定）
- `f`: 400kHz（I2C Fast Mode）

```
Rp_max = 1 / (0.8472 × 50pF × 400kHz)
       ≈ 59kΩ

実用的には立ち上がり時間 tr < 300ns を満たすため:
Rp_max ≈ 5.9kΩ
```

### 設計値: 4.7kΩ（標準E24系列）

**判定:**
- ✅ 967Ω < 4.7kΩ < 5.9kΩ → **範囲内**
- ✅ I2C Fast Mode (400kHz) 対応
- ✅ Standard Mode (100kHz) も対応
- ✅ 複数デバイス接続時も余裕あり

### バス容量の内訳（参考）
| 項目 | 容量 |
|------|------|
| SHT40 (I2C) | ~10pF |
| SCD41 (I2C) | ~10pF |
| 配線容量 (10cm) | ~15pF |
| Pi Pico GPIO | ~5pF |
| **合計** | **~40-50pF** |

## 6. ADC入力仕様

### Pi Pico ADC仕様
- **分解能**: 12bit SAR ADC
- **基準電圧**: VREF = 3.3V（内部レギュレータ）
- **入力範囲**: 0V 〜 3.3V
- **分解能**: 3.3V / 4096 = **0.8mV/LSB**
- **サンプリングレート**: 最大500ksps（通常10-100sps使用）

### 入力保護
- **過電圧**: 3.3Vを超える入力は**禁止**（ADC破損リスク）
- **5Vセンサー対応**: 外部分圧回路をユーザー側で実装推奨

#### 5V→3.3V分圧回路例
```
        5V Sensor Out
              │
              ├─── R1: 10kΩ
              │
ADC Input ────┤
              │
              ├─── R2: 20kΩ
              │
             GND

Vout = 5V × (20kΩ / (10kΩ + 20kΩ)) = 3.33V
```

### ターミナルブロックピン配列
```
[J5/J6/J7] 3ピンターミナル（5.08mm pitch）

Pin 1: GND    (黒線)
Pin 2: 3.3V   (赤線)
Pin 3: ADC    (黄線) → GP26/GP27/GP28
```

### 対応センサー例
| センサー | 出力 | 接続 | 備考 |
|----------|------|------|------|
| 容量式土壌水分 | 0-3.3V | 直接接続 | 秋月・Amazon汎用品 |
| 抵抗式土壌水分 | 抵抗値 | 分圧回路経由 | 要外部抵抗 |
| 日射センサー (PVSS-03) | 0-5V | **分圧回路必須** | 10kΩ+20kΩ推奨 |

## 8. 設計判断メモ

### 8.1 なぜGrove (HY2.0) コネクタか？
**理由:**
- M5Stackエコシステムとの互換性
- 殿の好み（CLAUDE.md Memory記載）
- 豊富なセンサーラインナップ（SHT40, SCD41, BH1750等）
- ケーブル着脱が容易（フィールド保守性）

**代替案とその却下理由:**
- JST PH (2.0mm): M5Stack非互換
- Qwiic (1.0mm): 小型だがケーブル入手性劣る
- ピンヘッダ: 誤配線リスク高、防水性低い

### 8.2 なぜターミナルブロックか？
**理由:**
- アナログセンサー（土壌水分、日射量）は配線自由度が必要
- ねじ止め式で接触不良が少ない
- フィールド環境で工具1本で交換可能
- 5.08mm pitchは配線作業が容易

**代替案とその却下理由:**
- Grove: ADCセンサーのGrove製品が少ない
- JST XH: ケーブル自作が必要、工具不要だが接触不良リスク

### 8.3 なぜ0603サイズか？
**理由:**
- **手はんだ可能**: 温度調整はんだごて + フラックスで実装可
- **小型**: 基板面積削減（Grove Shield相当サイズに収まる）
- **コスト**: 0402より安価、0805より省スペース
- **JLCPCB対応**: Basic部品で0603在庫豊富

**サイズ比較:**
| サイズ | 寸法 (mm) | 手はんだ | 備考 |
|--------|----------|---------|------|
| 0402 | 1.0 × 0.5 | 困難 | 拡大鏡必須 |
| **0603** | **1.6 × 0.8** | **可能** | **推奨** |
| 0805 | 2.0 × 1.25 | 容易 | 大きすぎ |
| 1206 | 3.2 × 1.6 | 容易 | スルーホール並 |

### 8.4 なぜGP16-21を完全予約するか？
**理由:**
- W5500 Ethernetコントローラが**ハードウェア固定**で使用
- 誤使用すると通信不能（デバッグ困難）
- CircuitPython/MicroPythonで自動初期化される
- ピンアサイン表で**視覚的に強調**（太字+警告）

**参照**: w5500_quickref.md（Memory MCP）

### 8.5 なぜI2C0/I2C1の2系統か？
**理由:**
- **アドレス競合回避**: 同一I2Cバス上で同じアドレスのデバイスは使用不可
- **バス容量分散**: 長距離配線時の容量負荷軽減
- **障害分離**: 片方のバスがハングしても他方は動作継続

**使用例:**
- I2C0 (J3): SHT40 (0x44) + SCD41 (0x62)
- I2C1 (J4): BH1750 (0x23)

### 8.6 Grove Shield for Pi Pico との違い
| 項目 | Grove Shield (市販品) | 本基板 |
|------|----------------------|--------|
| Digital GPIO | GP16/18/20使用 | **使用不可**（W5500競合） |
| I2C | 2ポート | 2ポート（同等） |
| UART | 2ポート | Reserved（将来拡張可） |
| ADC | 3ポート | 3ポート（同等） |
| 入手性 | ⚠️ 秋月販売終了 | ✅ JLCPCB自作 |
| W5500スタック | ❌ 非推奨（競合） | ✅ 最適化済み |

## 9. 1-Wire DS18B20 温度センサー

### 9.1 概要
DS18B20は、Dallas Semiconductor（現Maxim Integrated）製の1-Wire通信デジタル温度センサーである。土壌温度測定に最適。

### 9.2 主な仕様
- **通信**: 1-Wire（単線双方向）
- **分解能**: 9-12bit可変（0.0625°C @ 12bit）
- **測定範囲**: -55°C 〜 +125°C
- **精度**: ±0.5°C（-10°C 〜 +85°C）
- **電源**: 3.0V 〜 5.5V（3.3V動作可）
- **パラサイティック電源**: 対応（DQ線から給電可能）

### 9.3 ピンアサイン（TO-92パッケージ）
```
DS18B20 (Front View)
      ┌─┐
   1 ─┤ └─ Flat side
   2 ─┤
   3 ─┤
      └───
```

| Pin | 信号 | 接続先 |
|-----|------|--------|
| 1 | GND | Ground |
| 2 | DQ | GP2 (4.7kΩプルアップ → 3.3V) |
| 3 | VDD | 3.3V |

### 9.4 本基板実装（J8コネクタ）
```
[J8] 3ピンターミナル（5.08mm pitch）

Pin 1: GND    (黒線) → DS18B20 Pin1
Pin 2: DQ     (黄線) → DS18B20 Pin2 → GP2
Pin 3: 3V3    (赤線) → DS18B20 Pin3
```

### 9.5 プルアップ抵抗（R5）
- **値**: 4.7kΩ（標準値）
- **接続**: DQ (GP2) ─ R5 ─ 3.3V
- **理由**: 1-Wire通信はオープンドレイン方式のためプルアップ必須

### 9.6 複数センサー対応
1-Wireバスは最大数百デバイスを1本の信号線に接続可能。各DS18B20は固有の64bit ROMコードを持つ。

**使用例:**
- 土壌温度（深さ10cm）
- 土壌温度（深さ30cm）
- 空気温度（地上高1m）

MicroPython/CircuitPythonでROMアドレスを指定して個別読み取り可能。

### 9.7 MicroPython サンプルコード
```python
from machine import Pin
import onewire, ds18x20, time

# GP2を1-Wireバスとして初期化
ow = onewire.OneWire(Pin(2))
ds = ds18x20.DS18X20(ow)

# センサー検索
sensors = ds.scan()
print(f"Found {len(sensors)} DS18B20 sensors")

# 温度読み取り
while True:
    ds.convert_temp()
    time.sleep_ms(750)  # 12bit変換時間

    for rom in sensors:
        temp = ds.read_temp(rom)
        print(f"Sensor {rom.hex()}: {temp:.2f}°C")

    time.sleep(10)
```

## 10. ファン駆動回路（30mm 5V Fan）

### 10.1 概要
30mm角5Vファンを、Pi Pico GP22のPWM出力で駆動する回路。筐体内の強制換気により、センサー精度向上と熱対策を実現。

### 10.2 回路構成
```
        VSYS (5V)
            │
            ├─────────┬───────── J9 Pin1 (Fan +)
            │         │
           [Fan]      │
            │         │
            ├────[D1]─┤ (Flyback Diode 1N4148)
            │         │
            └─────────┴─ Q1 Drain (2N7002)
                      │
GP22 (PWM) ───[R6]─── Q1 Gate
                      │
                     [R7]
                      │
                     GND ───── J9 Pin2 (Fan -)

R6: 100Ω (ゲート抵抗)
R7: 10kΩ (プルダウン、起動時ファンOFF保証)
D1: 1N4148 (フライバック保護、逆並列接続)
```

### 10.3 部品選定理由

#### Q1: 2N7002 N-ch MOSFET
- **パッケージ**: SOT-23（手はんだ可能）
- **定格**: Vds=60V, Id=300mA
- **Rds(on)**: 約5Ω @ Vgs=4.5V
- **理由**: 30mm 5Vファン（100-150mA）に十分な余裕。低コスト。JLCPCB Basic在庫。

#### R6: ゲート抵抗 100Ω
- **目的**: GP22出力のサージ電流制限
- **計算**: 不要だが、ゲート容量充電時の突入電流を数十mA程度に制限
- **標準値**: 100Ω（E24系列）

#### R7: プルダウン 10kΩ
- **目的**: Pi Pico起動時のGP22フローティング防止
- **効果**: 起動時にゲート=0Vを保証し、ファン誤動作を防止
- **標準値**: 10kΩ（十分なプルダウン、消費電力も微小）

#### D1: フライバックダイオード 1N4148
- **目的**: ファンモーター停止時の逆起電力保護
- **接続**: ファン両端に逆並列（アノード=GND側、カソード=VSYS側）
- **定格**: 75V 200mA（ファン用途に十分）

### 10.4 PWM設定推奨値

#### PWM周波数: 25kHz
- **理由**: 可聴域外（人間の聴覚上限20kHz超）
- **効果**: PWM駆動時のキーン音を防止
- **Pi Pico対応**: MicroPython `PWM.freq(25000)` で設定可能

#### Duty Cycle範囲
- **0%**: ファン停止（R7プルダウンで確実にOFF）
- **20-30%**: 低速運転（静音、温度変化小）
- **100%**: 最大風量（センサー測定時、温度変化大きい時）

### 10.5 MicroPython サンプルコード
```python
from machine import Pin, PWM
import time

# GP22をPWM出力として初期化
fan_pwm = PWM(Pin(22))
fan_pwm.freq(25000)  # 25kHz (可聴域外)

# Duty Cycle設定（0-65535）
def set_fan_speed(percent):
    """ファン速度設定 (0-100%)"""
    duty = int(65535 * percent / 100)
    fan_pwm.duty_u16(duty)

# ソフトスタート（突入電流軽減）
def fan_soft_start():
    """ゆっくり起動（0% → 100% を2秒かけて）"""
    for i in range(0, 101, 5):
        set_fan_speed(i)
        time.sleep_ms(100)

# 使用例
fan_soft_start()        # ソフトスタート
time.sleep(10)
set_fan_speed(50)       # 50%運転
time.sleep(10)
set_fan_speed(0)        # 停止
```

### 10.6 消費電力試算

| 項目 | 電流 | 電圧 | 消費電力 |
|------|------|------|---------|
| ファン最大 | 150mA | 5V | 0.75W |
| センサー群 | 100mA | 3.3V | 0.33W |
| Pi Pico | 50mA | 3.3V | 0.17W |
| **合計** | - | - | **1.25W** |

**PoE給電（IEEE 802.3af）**: 15.4W供給 → **余裕十分**

### 10.7 ソフトスタート実装推奨理由
- **突入電流軽減**: ファン起動時の電源瞬断リスク低減
- **機械的寿命向上**: モーターへの衝撃緩和
- **PoE電源保護**: PoEスイッチの過電流保護誤動作防止

### 10.8 ファン仕様（参考）
- **サイズ**: 30mm × 30mm × 10mm
- **電圧**: DC 5V
- **電流**: 100-150mA（型番により異なる）
- **風量**: 5-8 CFM
- **ノイズ**: 20-25 dBA @ 100%
- **寿命**: 30,000時間（ボールベアリング）

### 10.9 動作確認手順
1. ファンコネクタ接続（J9: Pin1=赤=5V, Pin2=黒=GND）
2. MicroPythonで `set_fan_speed(100)` 実行
3. ファン回転確認（手で触れて風を感じる）
4. PWM周波数25kHz確認（オシロスコープ or 聴覚で無音確認）
5. `set_fan_speed(0)` でファン停止確認
