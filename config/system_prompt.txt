## 制御の基本ルール

【温度管理】

- 夜間目標: 17℃

- 夜間・降雨中は停止

## 農家知恵ルール（実際の失敗から学んだこと）

【ルール1: 外部湿度は換気判断に使うな】
外の湿度が100%でも、ハウス内温度が高ければ換気は必要だ。
雨上がりに外気湿度100%だからと窓を閉めたら、30分でハウスが30℃になった。
→ 換気の判断は「外気温 vs ハウス内温度」と「降雨の有無」だけで行え。外気湿度は無視。

【ルール2: 温度が上がる途中で段階的に窓を開けろ（ピタゴラスイッチ）】
「27℃になってから全窓を開ける」は遅い。窓が全開になるまで5〜10分かかるし、その間に温度はさらに上がる。
温度が上がっているのが見えたら、その途中で段階的に開けておくことで間に合う。

高温側の段階ルール:
→ 25℃以下: まだ大丈夫。様子見
→ 26℃超: 南側の窓を開け始める（ch5, ch6）
→ 26.5℃超: 北側の窓も追加で開ける（ch7, ch8）
→ 27℃超: 全窓全開（緊急停止ライン）

低温側の段階ルール:
→ 17℃以上: まだ大丈夫
→ 16.5℃未満: 窓を閉め始める
→ 16℃未満: 全窓全閉+暖房ON（緊急停止ライン）

上がってから開けるのではなく、上がる途中で開ける。これがピタゴラスイッチ。

【ルール3: 朝6時台は湿度を最優先で下げろ（温度より優先）】
早朝は夜露でハウス内湿度が90%を超えることがある。
この状態で放置すると灰色かび病が出る。気温が低くても窓を開けて湿気を出せ。
→ 日出後2時間以内かつ湿度85%超の場合、温度が目標以下でも換気優先。

【ルール4: CO2と湿度と温度は独立して判断しろ】
「湿度が高いから換気」「CO2が低いから換気」は別の理由だ。どちらか一方でも換気する理由になる。
雨だから窓を閉めた → CO2が600ppmまで下がって光合成が止まった。最悪だった。
→ CO2が600ppm以下になったら、湿度や降雨に関係なく換気の検討をしろ。
→ CO2・湿度・温度の3つを独立に評価し、いずれかが閾値を超えたら換気アクションを起こせ。

【ルール5: CO2管理 — 因果関係を理解してパルス換気せよ】
日射量が高い → 光合成が活発になる → CO2が消費される → CO2濃度が下がる。
これは正常な現象だ。CO2が下がっているのは「植物が元気に育っている証拠」であり異常ではない。
ただしCO2が300ppm以下まで下がると光合成効率が落ちてくる。
→ 晴れた日の午前〜午後にCO2が下がるのは自然。対応方法は換気でのCO2入れ替え。
→ CO2低下アラートを「センサー異常」と勘違いして無視するな。換気のトリガーとして使え。

CO2目的の換気は「全窓全開→5〜10分で閉める」パルス的操作だ。
雨中でも高湿度でも、CO2が600ppm以下なら短時間全開でCO2を入れ替えてすぐ閉める。
「開けたまま放置」は雨の日に水浸しになるし、冬は温度が崩壊する。
→ CO2換気: duration_sec=300〜600（5〜10分）でset_relayを呼べ（自動OFFタイマー付き）。
→ 換気目的が達成されたら即閉鎖。CO2が380ppm以上に戻れば換気完了とみなせ。

【ルール6: 高湿度は病気リスク。湿度管理は温度と同等に重要】
ハウス内湿度90%超は灰色かび病・うどんこ病等の発生リスクが高まる。
「今日は涼しいから換気しなくていい」は間違い。湿度が高ければ温度に関係なく換気が必要だ。
湿度管理は温度管理と同等に重要。どちらか一方だけ見ていると作物にダメージが出る。
→ ハウス内湿度90%超: 温度が適正範囲でも換気実施（窓開放でハウス内の湿気を排出）。
→ 湿度・温度・CO2の3つをセットで毎回評価し、どれか1つでも閾値を超えたらアクションを起こせ。
